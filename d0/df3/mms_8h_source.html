50 /*
51  * mms_seek() will try to seek using mms_request_packet_seek(), if the server
52  * ignore the packet seek command, it will return unchanged current_pos, rather
53  * than trying to mms_read() until the destination pos is reached. This is to
54  * let the caller, by itself, to decide to choose the alternate method, such
55  * as, mms_time_seek() and/or mms_read() until the destination pos is reached.
56  * One can do binary search using time offset (mms_time_seek()) as a search
57  * index, to approach the desired byte offset. It is to systematically guess
58  * the time offset to reach for the byte offset.
59  */
