138 // if mChunkSize is < 0 then this is the last chunk in the file and the actual length
139 // should be determined from the file size.
140 // The motivation for this is to allow writing the files without seeking to update size fields after every
141 // write in order to keep the file legal.
142 // The program can put a -1 in the mChunkSize field and
143 // update it only once at the end of recording.
144 // If the program were to crash during recording then the file is still well defined.

146 // 'pakt' Required if either/or mBytesPerPacket or mFramesPerPacket in the Format Description are zero
147 // For formats that are packetized and have variable sized packets.
148 // The table is stored as an array of one or two variable length integers.
149 // (a) size in bytes of the data of a given packet.
150 // (b) number of frames in a given packet.
151 // These sizes are encoded as variable length integers
152 
153 // The packet description entries are either one or two values depending on the format.
154 // There are three possibilities
155 // (1)
156 // If the format has variable bytes per packets (desc.mBytesPerPacket == 0) and constant frames per packet
157 // (desc.mFramesPerPacket != 0) then the packet table contains single entries representing the bytes in a given packet
158 // (2)
159 // If the format is a constant bit rate (desc.mBytesPerPacket != 0) but variable frames per packet
160 // (desc.mFramesPerPacket == 0) then the packet table entries contains single entries
161 // representing the number of frames in a given packet
162 // (3)
163 // If the format has variable frames per packet (asbd.mFramesPerPacket == 0) and variable bytes per packet
164 // (desc.mBytesPerPacket == 0) then the packet table entries are a duple of two values. The first value
165 // is the number of bytes in a given packet, the second value is the number of frames in a given packet
